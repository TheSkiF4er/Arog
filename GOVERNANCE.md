# Governance проекта Farog

Проект Farog развивается автором **TheSkiF4er** совместно с сообществом.

## Роли

- **Автор проекта (BDFL):** принимает финальные архитектурные решения, утверждает релизы.
- **Мейнтейнеры:** сопровождают конкретные подсистемы (runtime, cli, stdlib, docs).
- **Контрибьюторы:** отправляют pull request'ы, участвуют в обсуждениях, помогают с документацией и тестированием.

## Принятие решений

- Небольшие изменения могут обсуждаться и приниматься напрямую в pull request.
- Существенные изменения (архитектура, безопасность, формат конфигурации) требуют предварительного обсуждения в issue и, при необходимости, отдельного design-документа.
- В случае споров финальное решение остаётся за автором проекта.

## Релизы

- Минорные и патч-релизы выпускаются по мере накопления изменений.
- Мажорные релизы содержат значимые изменения API и требуют детального описания в `CHANGELOG.md` и документации.
