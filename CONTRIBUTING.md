# Руководство по участию в развитии Farog

Спасибо за интерес к Farog! Этот документ описывает, как вы можете помочь проекту.

## Принципы

- Уважение к другим участникам.
- Прозрачность изменений.
- Приоритет безопасности и стабильности.
- Максимальная уникальность решений: мы избегаем копирования чужих реализаций.

## Как начать

1. Форкните репозиторий и создайте отдельную ветку.
2. Найдите или создайте issue, описывающую задачу.
3. Разработайте решение, следуя правилам ниже.
4. Отправьте pull request и заполните шаблон.

## Технологический стек

- **Runtime (core):** C++ (CMake, C++20).
- **CLI и пакетный менеджер:** Go (go modules).
- **Стандартная библиотека и SDK:** TypeScript.

## Кодстайл

- C++: форматирование через `clang-format`, сборка через CMake, проверки `cmake --build` и по необходимости `ctest`.
- Go: форматирование `gofmt`, статический анализ `go vet`, тестирование `go test ./...`.
- TypeScript: форматирование и линт через `pnpm lint` и `pnpm format` (если используется).

Коммиты: осмысленные сообщения, по возможности один логический блок изменений на PR.

## Тесты

Перед отправкой PR обязательно:

```bash
# runtime (C++)
cmake -S runtime -B runtime/build
cmake --build runtime/build --config Debug
# при наличии тестов:
# ctest --test-dir runtime/build

# cli (Go)
cd cli
go test ./...
cd ..

# stdlib & packages (TS)
# pnpm install
# pnpm test
```

## Документация

Любое изменение публичного поведения Farog должно сопровождаться изменениями в документации (`docs/`, `examples/`, при необходимости — `README.md` и `CHANGELOG.md`).
